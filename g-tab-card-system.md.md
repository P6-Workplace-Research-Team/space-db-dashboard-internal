# 오피스 핵심 Index 탭 카드 시스템 정의 (v1.0)

## 1. 목적

본 문서는 오피스 핵심 Index 탭의 UI를 구성하는 카드 시스템을 정의한다.
지표 수가 많아도 UI를 단순하게 유지하기 위해 카드 유형을 제한하고, 모든 카드를 동일한 구조로 렌더링한다.

대시보드 카드 UI는 일반적으로 “헤더 + 본문 + 보조정보” 구조를 가지며 핵심 정보를 빠르게 전달해야 한다. ([Telerik.com][1])

---

## 2. 핵심 원칙

### 2.1 구조 통일

모든 카드는 아래 3영역 구조를 반드시 따른다.

```
[ 제목 영역: 지표 제목 + 유형 라벨 ]
[ 핵심값 영역 ]
[ 보조값 영역 ]
```

유형 라벨은 핵심값이 어떤 통계/판단 기준인지 알려주는 보조 제목이다.
모든 카드에 반드시 표시한다.

### 2.2 카드 타입 제한

카드 유형은 6개로 제한한다.
새로운 지표가 추가되어도 반드시 이 안에서 표현한다.

## 3. 카드 타입 정의

### 3.1 range-card (적정 범위형)

**용도**

* 적정 범위 (Q1 ~ Q3)
* 중앙값 + min/max

**구조**

```
제목
유형 라벨: 중앙값
중앙값
범위 (보조)
```

**예시**

* 한 층의 적정 면적
* 인당 면적

---

### 3.2 value-card (단일값)

**용도**

* 평균값
* 비율
* 개소

**구조**

```
제목
유형 라벨: 평균값 / 핵심값
대표값
설명(optional)
```

---

### 3.3 fixed-guide-card (고정 가이드)

**용도**

* 필터 영향을 받지 않는 설계 기준

**구조**

```
제목
유형 라벨: 권장값
권장값
조건/설명
```

**예시**

* 통로 너비
* 등간격
* 열 배치

---

### 3.4 top-choice-card (최다 선택형)

**용도**

* 가장 많이 사용되는 유형
* 가장 높은 비율

**구조**

```
제목
유형 라벨: 1순위
1위 값
보조 정보 (비율 / 평균값)
```

---

### 3.5 level-card (판단형)

**용도**

* 낮음 / 보통 / 높음 판단

**구조**

```
제목
유형 라벨: 판단값
레벨 (낮음/보통/높음)
실제 값
```

---

### 3.6 combo-card (복합값)

**용도**

* 개소 + 면적
* 2개 이상의 핵심값

**구조**

```
제목
유형 라벨: 복합값 / 평균값
값 1
값 2
```

## 4. 공통 UI 규칙

### 4.1 제목 영역

* 제목 영역은 지표 제목과 유형 라벨로 구성한다.
* 지표 제목은 최대 2줄까지 줄바꿈을 허용한다.
* 제목과 유형 라벨 사이에 인위적인 고정 높이나 큰 여백을 두지 않는다.
* 짧은 제목도 유형 라벨이 바로 아래에 붙어 보여야 한다.
* 지표 제목은 유형 라벨보다 명확히 눈에 띄어야 한다.
* 구현 기준: 제목 `16px / 500 / #868686`, 유형 라벨 `10px / 500 / #a7a7a7`.
* 제목 크기는 기존 11px 기준에서 약 5px 키운다.

### 4.2 유형 라벨

* 모든 카드에 표시한다.
* 핵심값의 통계/판단 유형을 설명한다.
* 지표 제목 바로 아래에 배치한다.
* 하단 보조값 영역에 중복 표시하지 않는다.
* 시각적으로 제목보다 약하게 보이되, 읽을 수 있어야 한다.

**대표 라벨**

* 중앙값
* 평균값
* 평균 비중(%)
* 권장값
* 1순위
* 핵심값
* 판단값
* 복합값

**구현**

```html
<header class="gm-title">업무 공간 평균 면적 비중</header>
<div class="gm-type-label">평균값</div>
```

```css
.gm-title {
  font-size: 16px;
  font-weight: 500;
  color: #868686;
  letter-spacing: 0;
}

.gm-type-label {
  font-size: 10px;
  font-weight: 500;
  color: #a7a7a7;
}
```

```js
guideHeroShell({ title, label, valueHtml })
guideListShell({ title, label, valueHtml })
guideRangeShell({ title, label, mainHtml })
```

* `valueLabel`이 있으면 그 값을 유형 라벨로 사용한다.
* `valueLabel`이 없고 퍼센트 지표이면 `평균값`을 기본 라벨로 사용한다.
* 범위형 카드는 `중앙값`을 기본 라벨로 사용한다.

### 4.3 핵심값 영역

* 카드에서 가장 강조
* 중앙 정렬
* 가장 큰 폰트

### 4.4 단위 표기

* 단위는 핵심값보다 작은 보조 글자 스타일로 표시한다.
* 단위는 위첨자/아래첨자처럼 띄우거나 내리지 않는다.
* 핵심값과 단위의 하단 라인이 동일하게 보이도록 정렬한다.
* 구현 기준: 값과 단위를 `inline-flex`로 묶고 `align-items: flex-end`를 적용한다.
* 단위는 `line-height: 1`, `vertical-align: bottom`, `position: static`을 기본으로 한다.

### 4.5 보조값 영역

* 카드 하단 고정
* 좌/우 정렬 가능
* 색상 약하게

## 5. 금지 사항

❌ 카드마다 다른 레이아웃 사용
❌ 지표마다 CSS 따로 정의
❌ 카드 높이 제각각
❌ 중앙값 위치가 카드마다 다름

## 6. 구현 원칙

### 6.1 데이터 기반 렌더링

```js
{
  type: "range",
  title: "한 층의 적정 면적",
  main: 271.75,
  unit: "평",
  min: 164.32,
  max: 416.26
}
```

### 6.2 단일 렌더러

```js
renderCard(metric)
```

* metric.type에 따라 내부 UI만 분기
* HTML 구조는 동일 유지
