# 오피스 핵심 Index 탭 카드 시스템 정의 (v1.0)

## 1. 목적

본 문서는 오피스 핵심 Index 탭의 UI를 구성하는 카드 시스템을 정의한다.
지표 수가 많아도 UI를 단순하게 유지하기 위해 카드 유형을 제한하고, 모든 카드를 동일한 구조로 렌더링한다.

대시보드 카드 UI는 일반적으로 “헤더 + 본문 + 보조정보” 구조를 가지며 핵심 정보를 빠르게 전달해야 한다. ([Telerik.com][1])

---

## 2. 핵심 원칙

### 2.1 구조 통일

모든 카드는 아래 3영역 구조를 반드시 따른다.

```
[ 제목 영역 ]
[ 핵심값 영역 ]
[ 보조값 영역 ]
```

### 2.2 카드 타입 제한

카드 유형은 6개로 제한한다.
새로운 지표가 추가되어도 반드시 이 안에서 표현한다.

---

## 3. 카드 타입 정의

### 3.1 range-card (적정 범위형)

**용도**

* 적정 범위 (Q1 ~ Q3)
* 중앙값 + min/max

**구조**

```
제목
중앙값 (핵심)
범위 (보조)
```

**예시**

* 한 층의 적정 면적
* 인당 면적

---

### 3.2 value-card (단일값)

**용도**

* 평균값
* 비율
* 개소

**구조**

```
제목
대표값
설명(optional)
```

---

### 3.3 fixed-guide-card (고정 가이드)

**용도**

* 필터 영향을 받지 않는 설계 기준

**구조**

```
제목
권장값
조건/설명
```

**예시**

* 통로 너비
* 등간격
* 열 배치

---

### 3.4 top-choice-card (최다 선택형)

**용도**

* 가장 많이 사용되는 유형
* 가장 높은 비율

**구조**

```
제목
1위 값
보조 정보 (비율 / 평균값)
```

---

### 3.5 level-card (판단형)

**용도**

* 낮음 / 보통 / 높음 판단

**구조**

```
제목
레벨 (낮음/보통/높음)
실제 값
```

---

### 3.6 combo-card (복합값)

**용도**

* 개소 + 면적
* 2개 이상의 핵심값

**구조**

```
제목
값 1
값 2
```

---

## 4. 공통 UI 규칙

### 4.1 제목 영역

* 항상 동일 높이 유지
* 2줄 기준
* 줄바꿈 허용

### 4.2 핵심값 영역

* 카드에서 가장 강조
* 중앙 정렬
* 가장 큰 폰트

### 4.3 보조값 영역

* 카드 하단 고정
* 좌/우 정렬 가능
* 색상 약하게

---

## 5. 금지 사항

❌ 카드마다 다른 레이아웃 사용
❌ 지표마다 CSS 따로 정의
❌ 카드 높이 제각각
❌ 중앙값 위치가 카드마다 다름

---

## 6. 구현 원칙

### 6.1 데이터 기반 렌더링

```js
{
  type: "range",
  title: "한 층의 적정 면적",
  main: 271.75,
  unit: "평",
  min: 164.32,
  max: 416.26
}
```

### 6.2 단일 렌더러

```js
renderCard(metric)
```

* metric.type에 따라 내부 UI만 분기
* HTML 구조는 동일 유지

---

## 7. 기대 효과

* 카드 30개 → UI 6개로 축소
* 유지보수 비용 감소
* 디자인 일관성 확보
* Codex 수정 효율 극대화
