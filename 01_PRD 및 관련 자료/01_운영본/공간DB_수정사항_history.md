# 공간DB 수정사항 History

작성 목적: 공간DB HTML 대시보드, 계산식, UI/UX, PRD 반영 이력을 날짜별로 누적 기록한다.

운영 원칙

- 이 파일은 누적형 단일 history 파일이다.
- 같은 날 여러 작업을 해도 새 history 파일을 만들지 않고 아래에 계속 추가한다.
- 계산식, 필터 구조, 지표 정의, 시각화 원칙처럼 운영 기준에 영향을 주는 변경을 우선 기록한다.
- PRD는 별도 버전 파일로 관리하고, 이 history는 변경 배경과 작업 내역을 추적하는 용도로 사용한다.

## 16. 2026-04-13 바로가기/소셜/지원공간 UI 오류 수정

- 우측 `바로가기` 패널에서 항목 hover 시 해당 카드만 미리 강조되고, 클릭 시 해당 카드가 보라색 테두리로 잠시 강조되도록 동작을 보강했다.
- `바로가기` 생성 순서를 초기 배치 직후로 앞당겨, 후반 탭 렌더 오류가 나더라도 목차가 비어 보이지 않도록 정리했다.
- 소셜공간 탭에서 `renderELoungeChart()`가 비정상 상태로 남아 있던 문제를 복구했다.
- 이 오류로 인해 `라운지 평균 면적`, `면적 구간별 라운지 유형`, `라운지 유형별 평균 면적`, `라운지 내 세부 프로그램 선택 비율` 카드가 비거나, 이어서 지원공간 전체 렌더가 중단되던 문제를 함께 수정했다.
- 소셜공간 상단 KPI의 `라운지 평균 면적`, `면적 구간별 라운지 유형` 값을 다시 연결했다.
- 라운지 유형별 평균 면적은 실제 `라운지` 공간을 보유한 도면만 기준으로 집계되도록 보정했다.
- 지원공간 탭은 소셜 탭 예외로 인해 연쇄적으로 렌더가 멈추던 문제를 해소해 전체 지표가 다시 노출되도록 복구했다.

## 17. 2026-04-13 소셜공간 값형 차트/라벨 정리

- PRD 원칙에 맞춰 `개소/면적 같은 값형 추이 = 꺾은선 차트`, `비율형 추이 = 바 차트` 규칙을 재확인했다.
- 소셜공간의 `인원 구간별 탕비 평균 면적`, `가용 면적 구간별 탕비 평균 면적`은 단위 토글과 관계없이 항상 꺾은선 차트로 유지되도록 수정했다.
- 기존에는 `평 → m² → 평` 토글 시 예전 막대 그래프 렌더 함수가 재호출되어 표현 방식이 뒤바뀌던 문제가 있었다.
- `라운지 내 세부 프로그램 선택 비율` 카드의 세부 프로그램 라벨이 `??`로 보이던 문제를 수정했다.
- 표시 라벨을 `탕비`, `카페`, `엔터`, `휴게`, `이벤트`, `업무(오픈)`, `업무(룸)`, `폰부스`로 정리했다.
- 위 수정 사항은 브라우저 캡처 기반으로 소셜공간/지원공간 탭 실제 렌더 결과를 확인하며 검증했다.

## 18. 2026-04-13 단위 표기 및 차트 단위 노출 정리

- 화면 내 `m2`, `m²` 표기를 모두 `㎡`로 통일했다.
- 버튼 라벨, KPI 보조 표기, 카드 단위 표기, 차트 축 라벨에서 `평`, `㎡`, `개소`, `%`, `건`, `인`이 일관되게 보이도록 정리했다.
- 값형 선형 차트의 데이터라벨에도 단위가 함께 표시되도록 보강했다.
- 특히 `연도별 회의실 평균 개소`, `인원 구간별 회의실 평균 개소`, `면적 구간별 회의실 평균 개소`, 지원공간의 평균 개소 차트에서 `개소`가 함께 보이도록 수정했다.
- 공통 차트 툴팁 포맷을 정리해 `y축 단위`를 읽어 `개소`, `평`, `㎡`, `%`가 툴팁에도 같이 표시되도록 맞췄다.
- 위 변경은 브라우저 캡처로 실제 렌더 화면을 확인하며 검증했다.

## 19. 2026-04-15 G탭(설계 가이드) 신설 — 탭 순서 및 기본 레이아웃

- 활성 스텝은 보라 배경(`#eae5fe`)으로 강조하고, 이미 지난 스텝은 투명도를 낮춘다.

## 21. 2026-04-15 G탭 필터 구조 개편

- `현재 설계 기준` 패널을 제거하고, 필터 조건 패널 1개 구조로 단순화했다.
- 기본 필터는 `가용면적구간`, `인원구간`, `업종` 3개로 유지하고, 나머지 변수는 `추가 필터 보기` 버튼으로 접고 펼 수 있도록 했다.
- 필터 UI를 단일 `<select>` 드롭다운에서 토글 칩(pill) 방식으로 교체해 다중 선택을 지원한다.
- `전체` 칩은 해당 필터를 초기화하며, 나머지 칩은 클릭할 때마다 선택/해제가 토글된다.
- 동일 필드 내 다중 선택은 OR로 동작한다(PRD 2.3 핵심 제약 조건 준수).
- 필터링 결과가 30건 미만인 경우 `현재 N건 미만의 도면을 대상으로 분석중입니다. 이는 모수가 적어 해석에 유의가 필요합니다.` 경고 배너를 자동 표시한다.

## 23. 2026-04-15 성능 최적화

- 파일 크기를 1,406KB에서 914KB로 줄였다(-34%).
- 주요 최적화 내용은 다음과 같다.
  - DRAWINGS 데이터: 네임드 오브젝트 배열(`{key: value, …}`)을 헤더+행 형식으로 압축해 357KB → 136KB로 줄였다.
  - SPACES 데이터: 문자열 카테고리 컬럼(프로그램, 세부유형, 오픈실 등)을 정수 인덱스로 인코딩해 891KB → 629KB로 줄였다. 항상 1이었던 CNT 컬럼도 제거했다.
  - 레이지 탭 렌더링: 필터 변경 시 현재 활성 탭만 즉시 렌더하고, 나머지 탭은 해당 탭 진입 시 렌더하도록 변경했다. 기존에는 필터를 바꿀 때마다 6개 탭의 차트 29개가 전부 재렌더됐다.
  - 중복 `renderGuideTab` 함수 정의 2개를 제거했다(코드베이스 누적 중복).
  - CDN `preconnect` 및 `dns-prefetch` 힌트를 추가했다.
  - Pretendard 폰트를 논블로킹(`media="print"` → onload) 방식으로 로드해 렌더 블로킹을 제거했다.
