## 1. 제품 개요

### 1.1 배경 및 목적

퍼시스·퍼플식스 스튜디오는 2002년부터 현재까지 수집·축적한 540건의 오피스 도면 데이터(2025 공간DB 기준)를 보유하고 있다. 기존에는 이 데이터를 Tableau 기반 대시보드로 제공했으나, 영업·제안 담당자들이 다음과 같은 불편을 지속적으로 제기했다.

- 숫자만 나열된 데이터를 직관적으로 해석하기 어렵다.
- 자신의 프로젝트 조건(인원, 면적, 산업군, 권역 등)에 맞는 지표만 골라서 보고 싶다.

본 PRD는 이러한 Pain Point를 해소하기 위해, 실제 데이터셋 기반 수치를 그대로 유지하면서 조건별 필터링과 직관적인 시각화를 제공하는 공간DB 대시보드의 요구사항을 정의한다.

### 1.2 제품 목표

- 오피스 설계 프로젝트 착수 시 적정 회의실 개소, 면적, 인당 면적 등 핵심 수치를 빠르게 참고할 수 있어야 한다.
- 연도별 오피스 공간 변화 트렌드를 세미나, 리포트, 연구 자료로 활용할 수 있어야 한다.
- Tableau 수준의 계산 정밀도를 유지하되, 연구팀 지원 없이도 실무자가 직접 활용할 수 있는 UX를 제공해야 한다.
- 필터 조합에 따라 필요한 지표를 빠르게 탐색할 수 있어야 한다.
- 대시보드는 매년 1회 업데이트되며, 데이터셋은 CSV/XLSX 기반으로 제공된다.

## 2. 범위 및 제약 조건

### 2.1 분석 데이터베이스 설명

공간DB는 크게 다음 세 데이터 소스를 결합해 구성된다.

- A-1: 도면 내 세부 공간의 면적 및 공간 프로그램 분류 데이터셋
- A-2: 업무/임원 공간의 인원 및 가구 분류 데이터셋
- B: 도면 단위의 메타데이터 데이터셋

결합 키는 `파일이름`이며, 지표 산출 시 기준은 다음 원칙을 따른다.

- `네모 기준`: 개별 공간(네모) 단위로 평균, 면적, 개수를 계산한다.
- `도면 기준`: 동일 `파일이름`을 갖는 도면 단위로 평균, 비율, 존재 여부를 계산한다.

### 2.2 주요 운영 필드

운영상 자주 사용하는 주요 필드는 다음과 같다.

- 프로그램
- 세부유형
- 오픈실
- 데스크형태
- 데스크너비
- 가용면적
- 가용면적구간
- 인원수
- 인원구간
- 연도
- 주소
- 권역
- 산업군
- 사업장 유형
- 참고도면
- 프라임오피스
- 대기업여부
- 소속대기업

### 2.3 핵심 제약 조건

- 모든 수치는 실제 데이터셋 기반 계산 결과만 사용한다.
- Tableau와 HTML 대시보드는 동일한 계산 원칙을 유지해야 한다.
- 필터는 동일 필드 내 다중선택 OR, 서로 다른 필드 간 조합 AND를 기본 원칙으로 한다.
- 외부 공유 시 사내 자료 성격을 고려한 권한 관리와 출처 표기 안내가 필요하다.

## 3. 공통 계산 원칙

### 3.1 공통 원칙

- `파일이름`은 도면 단위 고유값으로 취급한다.
- `네모이름`은 개별 공간 단위 고유값으로 취급한다.
- `NULL 네모`는 실제 공간으로 간주하지 않는다.
- 평균 개소는 지표 정의에 따라 `전체 도면 기준` 또는 `해당 공간이 존재하는 도면만 평균 포함` 여부를 명시한다.

### 3.2 비율형 지표 원칙

- 단순 존재 비율: `조건을 만족하는 도면 수 / 기준 도면 수`
- 공간 프로그램 구성 비율: 도면별 비율을 먼저 계산한 뒤, 이를 평균한다.
- 100% 구성비 차트는 누적 막대 또는 누적 가로 막대로 표시한다.

### 3.3 소셜/회의/지원 특수 원칙

- 라운지 계획 비율은 `세부유형=라운지` 네모를 1개 이상 가진 도면 기준으로 계산한다.
- 회의실 관련 지표는 `기타회의공간`을 제외한다.
- 지원공간 평균 개소는 `해당 공간이 존재하는 도면만 평균 포함` 기준을 기본으로 한다.
- 회의실 평균 개소는 `회의실이 존재하는 도면만 평균 포함` 기준을 유지한다.

## 5. 탭별 지표 구조

### 5.0 G탭: 설계 가이드 (오피스는 어떻게 만들어야 하나요?)

G탭은 공간DB의 지표를 실제 설계 순서에 맞춰 재배치한 가이드 탭이다. 내비게이션 첫 번째 탭으로 배치해 메인 화면으로 사용한다.

#### 5.0.1 레이아웃 원칙

- G탭 진입 시 좌측 필터 사이드바를 숨기고, 콘텐츠 영역이 전폭을 사용한다.
- 우측 바로가기는 `① 전체 조닝`, `② 업무공간` 등 단축 형식으로 표시한다.
- 스텝 내비게이터는 `01 전체 조닝 › 02 업무 › … › 06 지원` 형태의 1행 흐름으로 배치한다.

#### 5.0.2 상단 필터 구조

G탭은 별도의 상단 조건 필터를 갖는다. 사이드바 필터와 AND로 교차 적용된다.

- 기본 필터: 가용면적구간, 인원구간, 업종 (3개)
- 추가 필터: 기본 필터 외 나머지 전체 필터 변수 (접기/펼치기)
- 필터 UI: 토글 칩(pill) 방식, 동일 필드 내 다중 선택 OR
- 전체 칩: 해당 필드 필터 초기화
- 경고 배너: 필터링 결과가 30건 미만이면 자동 표시

#### 5.0.3 설계 스텝 구조

각 스텝은 접기/펼치기가 가능하며, 전체보기 모드에서는 모든 스텝이 동시에 펼쳐진다. 스텝별 지표 카드는 항상 1행으로 나란히 표시한다.

| 스텝 | 제목 | 주요 지표 |
|------|------|-----------|
| 1 | 전체 조닝 | 가용면적 범위, 인원 범위, 공간 프로그램 비중 |
| 2 | 업무공간 | 업무 비중, 인당 면적, 주요 데스크 형태/너비 |
| 3 | 임원공간 | 임원 비중, 주요 가구 구성, 평균 면적 |
| 4 | 회의공간 | 회의 비중, 규모별 평균 개소 및 면적 |
| 5 | 소셜공간 | 소셜 비중, 라운지 조성 비율, 탕비 면적 |
| 6 | 지원공간 | 지원 비중, 폰부스+선택업무공간 개소 |

#### 5.0.4 계산 원칙

- 모든 지표는 필터링된 도면 집합을 기준으로 실시간 계산한다.
- 비중은 `도면별 해당 프로그램 면적 / 가용면적` 평균으로 산출한다.
- 개소형 지표는 `해당 공간이 존재하는 도면만 평균 포함` 기준을 따른다.
- 면적형 지표는 m² 기준 계산 후 필요 시 평 환산(× 0.3025)한다.

#### 5.0.5 오피스 핵심 INDEX UI/UX 운영 규칙

- 오피스 핵심 INDEX는 설계 단계별 핵심 지표를 한 화면에서 빠르게 비교하는 데스크탑 우선 탭으로 운영한다.
- 데스크탑 뷰에서는 모니터 높이가 달라도 탭 내부 세로 스크롤 없이 핵심 정보가 1화면 안에 보이도록 viewport-fit 축소 로직을 적용한다.
- viewport-fit 축소 기준점은 좌상단으로 두어, 특정 해상도에서 우측 카드가 잘리지 않도록 한다.
- 리사이즈 시 레이아웃 재계산은 즉시 반응해야 하며, 불필요한 중복 resize/observer 호출은 두지 않는다.

- 스텝 구조는 STEP n / 단계명을 상단 핵심 헤더로 사용하고, 질문형 문장은 그 아래 보조 텍스트로 둔다.
- 상단 스텝 네비게이터의 단계명 색상은 각 스텝의 상징 색상을 사용한다.
- 스텝 설명형 본문은 기본 화면에서 제거하고, 스텝명과 핵심 질문만 남긴다.

- 전체 조닝의 범위형 지표 카드는 지표명 / 중앙값 / 최소 / 최대 구조를 기본으로 한다.
- 범위형 지표의 구분자는 `-`를 사용하며, 최소값과 최대값의 정중앙에 오도록 정렬한다.
- 1개층의 적정 인원 수는 범위형 카드로 표시하며 중앙값/최소/최대 모두 자연수 반올림 값으로 표기한다.

- 면적 관련 지표는 기본 계산을 m² 기준으로 유지하되, 화면 표시 단위는 평 / m² 토글로 전환한다.
- 단위 토글은 카드 전체 재렌더링이 아니라 값 텍스트만 즉시 갱신하는 방식으로 반응해야 한다.
- 평, m², 평/인 등 단위 표기는 핵심 숫자보다 작은 크기로 표시한다.

- 일반 지표 카드와 범위형 카드, 프로그램 비중 카드는 동일한 카드 셸, 여백, 제목 위계를 사용한다.
- 다른 스텝의 일반 지표 카드도 전체 조닝 카드와 같은 디자인 시스템 안에서 보이도록 통일한다.
- 핵심 값 타이포는 화면 크기에 따라 함께 반응하도록 clamp 기반 반응형 크기 규칙을 사용한다.

- 프로그램 비중 카드는 라벨이 아닌 퍼센트 값을 핵심 값으로 취급한다.

- 오피스 핵심 INDEX 지표 카드는 다음 5개 유형으로 구분한다.
  - 범위형 카드: 중앙값을 핵심 값으로 강조하고, 최소/최대는 같은 카드 안에서 더 작은 회색 보조 값으로 표시한다.
  - 강조 %형 카드: 전체 비중이나 조성 비율처럼 퍼센트가 핵심인 지표에 사용하며, 값 라벨은 `%`로 표기한다.
  - 단순 값형 카드: 개소, 대표 구성, 대표 면적처럼 핵심 값 1개를 직접 보여주는 지표에 사용한다.
  - 설명형 카드: 값 1개와 함께 보조 설명, 기준 문장, 예외 조건을 같이 보여줘야 하는 지표에 사용한다.
  - 리스트형 카드: 프로그램별 비중, 대표 데스크 형태/너비, 권장 통로처럼 항목 라벨과 값 쌍이 2개 이상 반복되는 지표에 사용한다.

- 모든 카드 유형은 동일한 카드 셸, 제목 위계, 내부 여백 규칙을 공유한다.
- 핵심 값은 기본적으로 한 줄 노출을 우선하며, 카드 폭을 넘지 않도록 clamp 기반 타이포 또는 카드 유형 전용 축소 규칙을 사용한다.
- 단위 표기(`평`, `㎡`, `인`, `개소`, `평/인` 등)는 핵심 숫자보다 작은 크기와 낮은 강조도로 표시한다.
- 범위형 카드의 최소/최대 값은 중앙값보다 덜 강조되도록 더 작은 폰트와 회색 계열 색상을 사용한다.
- 리스트형 카드에서 보조 설명(예: 선택 비율, 최소 기준)은 핵심 값 아래의 보조 텍스트로 처리한다.
- 리스트형 카드 중 권장 주통로/부통로 너비처럼 수치 길이가 긴 카드는 좌측 정렬과 전용 축소 규칙을 허용한다.
- 설명형 카드 중 권장 등간격 너비처럼 가이드 문장을 함께 노출하는 카드는 값 1줄, 보조 설명 1~2줄 구조를 기본으로 한다.
- 업무 탭에서 정리한 카드 유형 규칙은 임원/회의/소셜/지원 스텝에도 동일하게 적용해, 스텝이 달라도 같은 의미의 지표는 같은 카드 유형으로 보이도록 유지한다.
- 프로그램 비중 카드의 상단 보조 라벨은 값 대신 %를 사용한다.
- 프로그램 비중 카드에는 프로그램명과 퍼센트를 함께 노출하되, 퍼센트 값은 다른 핵심 지표 값과 동일한 강조 수준으로 표시한다.

### 5.1 개괄

- 인원 구간별 도면 수
- 가용 면적 구간별 도면 수
- 인원 구간별 가용 면적 범위
- 가용 면적 구간별 인원 범위
- 연도별 오피스 공간 프로그램 구성 비율

### 5.2 업무공간: 일반

- 업무 공간 비중
- 평균 인당 면적
- 주요 등간격(일자형/L자형)
- 주요 데스크 형태
- 주요 데스크 너비
- 자율좌석제 시행 비중
- 데스크 형태별 사용 비중
- 데스크 너비별 사용 비중
- 팀장석 구성 유형
- 그룹업무공간 보유 비중

### 5.3 업무공간: 임원

- 임원석 최다 구성(실형)
- 임원실 평균 면적(실형)
- 오픈 임원석 보유 도면 비율
- 임원 가구 구성 비율
- 임원 가구 구성별 평균 면적

운영 기준

- 오픈 임원석 보유 도면 비율은 `임원공간을 보유한 도면 중 오픈 임원석 네모를 가진 도면 비율`로 계산한다.
- 전체 구성 비율은 실형/오픈형 각각 도넛 차트로 본다.
- 연도별 구성 비율은 실형/오픈형 각각 바 차트로 본다.

### 5.4 회의공간

- 회의실 평균 개소
- 인원 구간별 회의실 평균 개소
- 면적 구간별 회의실 평균 개소
- 회의실 규모별 평균 개소 및 평균 면적
- 회의실 가구 스타일 비중
- 회의실 규모별 가구 스타일 비중
- 오픈 회의 비중

운영 기준

- `조성/보유/비중` 같은 비율형 추이는 바 차트로 본다.
- `개소/면적` 같은 값형 추이는 꺾은선 차트로 본다.

### 5.5 소셜공간

- 라운지 계획 비율
- 연도별 소셜공간 유형 구성 비율
- 연도별 라운지 계획 여부
- 인원 구간별 탕비 평균 면적
- 가용 면적 구간별 탕비 평균 면적
- 라운지 유형별 평균 면적
- 라운지 내 세부 프로그램 선택 비율

운영 기준

- 라운지 계획 비율은 실제 라운지 네모 기준으로 계산한다.
- 탕비 평균 면적은 `세부유형=탕비` 네모의 평균 면적을 해당 도면의 인원구간/가용면적구간 기준으로 본다.
- 라운지 내 세부 프로그램 선택 비율에서 `커뮤니티 오픈`은 비교 대상에서 제외한다.

### 5.6 지원공간

- 폰부스+선택업무공간_실 조성 비율
- 폰부스+선택업무공간_실 평균 개소
- 연도별 폰부스+선택업무공간_실 조성 비율
- 인원 구간별 폰부스+선택업무공간_실 평균 개소
- 가용 면적 구간별 폰부스+선택업무공간_실 평균 개소
- OA 평균 개소

운영 기준

- 조성 비율은 바 차트로 본다.
- 개소/면적은 꺾은선 차트로 본다.

## 6. 공통 시각화 원칙

### 6.1 차트 타입 원칙

- 비율형 추이: 바 차트
- 개소/면적 추이: 꺾은선 차트
- 전체 구성 비율 비교: 도넛 또는 누적 막대
- 순위형 비교: KPI 또는 카드형 리스트

### 6.2 Y축 원칙

- 단일 퍼센트 추세 차트는 `percentAxisAuto()` 기준으로 자동 조정한다.
- 다만 100% 누적 구성비 차트는 0~100 축을 유지한다.
- 값 자체가 중요한 차트는 데이터라벨이 직접 노출되어야 한다.

### 6.3 탐색 UI 원칙

- 좌측은 고정 필터 패널을 사용한다(G탭 제외).
- 우측은 세로 타임라인형 바로가기를 사용한다.
- 바로가기는 현재 보고 있는 카드에 따라 자동 하이라이트된다.
- 카드 앵커 이동 시 헤더에 제목이 가려지지 않도록 오프셋을 적용한다.

### 6.4 디자인 시스템 원칙 (v2.0)

- 폰트: Pretendard
- 메인 퍼플: `#7800f5`
- 카드 border-radius: 4px
- 버튼 border-radius: 0px (직선 기조)
- 테두리: `#e8e8e8` 뉴트럴 그레이
- 박스섀도: `0 2px 8px rgba(0,0,0,0.06)` 이하
- G탭 스타일은 `#sec-G` 스코프로 격리해 기존 탭에 영향을 주지 않는다.

## 7. 비기능 요구사항

- 데이터 무결성: 실제 데이터셋 기반 수치만 사용
- 계산 일관성: Tableau와 동일한 기준 유지
- 응답성: 필터 적용 후 현재 탭 지표가 즉시 갱신되어야 함. 비활성 탭은 진입 시 렌더한다.
- 사용성: 데스크탑 우선 구조, 명확한 필터/탐색 흐름 제공
- 유지보수성: history와 PRD 버전 파일을 분리해 관리
- 파일 크기: 단일 HTML 파일 기준 1MB 이하를 목표로 한다.
