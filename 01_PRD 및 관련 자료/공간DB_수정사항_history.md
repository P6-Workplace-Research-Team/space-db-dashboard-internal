# 공간DB 수정사항 History

작성일: 2026-04-07  
작성 목적: 공간DB HTML 대시보드, Tableau 기준, PRD 개정 사항을 한 문서에서 추적하기 위한 변경 이력 기록

## 1. 문서 목적

본 문서는 2026-04-07 기준으로 공간DB 대시보드에서 수행한 계산식 정비, UI/UX 개편, 필터 구조 변경, 문서 개정 사항을 기록한다.  
원칙은 다음과 같다.

- Tableau 대시보드와 HTML 대시보드의 산출 기준을 최대한 동일하게 맞춘다.
- PRD에 없거나 모호했던 운영 기준은 실제 합의된 방식으로 명문화한다.
- 향후 운영자가 변경 이력을 빠르게 추적할 수 있도록 원인, 수정 내용, 최종 기준을 함께 남긴다.

## 2. 2026-04-07 1차 정비

### 2.1 계산식 및 산출 기준 정리

- HTML 대시보드와 Tableau 워크북의 지표별 계산 방식을 대조 검토
- 회의, 소셜, 지원, 개괄, 업무 지표의 분모/분자 기준 재정의
- 연도 추이형 지표의 집계 방식을 `연도 합산 후 비율`에서 `도면별 비율 평균`으로 수정
- 회의 지표에서 `기타회의공간` 제외 기준 통일
- 지원공간 평균 개소는 `해당 공간이 존재하는 도면만 평균 포함`으로 정리
- 연도 추이 시각화는 `2020~2025`만 표시
- 모든 숫자 표시는 소수점 둘째 자리까지 통일

### 2.2 필터 구조 개편

- 단일 선택 드롭다운 중심 구조를 다중 선택 체크박스 기반 필터 UI로 변경
- 필터를 다음 3개 카테고리로 재구성
- 도면 관련 필터링: `연도`, `가용면적구간`, `인원 구간`, `프라임 오피스`, `참고도면`
- 위치 관련 필터: `주소`, `권역`
- 기업 관련 필터: `산업군`, `사업장 유형`, `대기업 여부`, `소속 대기업`
- 화면 라벨 `업종`을 `산업군`으로 변경
- `참고도면`, `대기업 여부`, `소속 대기업`, `프라임오피스` 필터 추가
- `프라임오피스`는 `O`, `X`, `사옥` 3개 값을 각각 별도 옵션으로 노출
- 동일 필드 내 다중 선택은 OR, 서로 다른 필드 조합은 AND 로 동작하도록 수정

### 2.3 레이아웃 및 탐색 UX 개편

- 상단 고정 필터 바를 폐지하고 좌측 고정 사이드 필터 패널로 변경
- 필터 패널 접기/펼치기 기능 추가
- 탭 네비게이션을 floating 구조에서 헤더 내 고정 구조로 이동
- 우측 바로가기 패널 추가
- 바로가기 클릭 시 지표 제목이 헤더에 가려지지 않도록 scroll offset 적용
- 스크롤 위치에 따라 현재 보고 있는 카드가 우측 바로가기에서 자동 하이라이트되도록 수정

### 2.4 개괄 탭 표현 방식 변경

- `인원 구간별 도면 수`, `가용 면적 구간별 도면 수`를 그래프에서 자연수 기반 히트맵 표로 변경
- `인원 구간별 가용 면적 범위`, `가용 면적 구간별 인원 범위`는 박스플롯과 함께 표를 제공하는 구조로 개편
- 표에는 `중앙값`, `적정 최소값(Q1)`, `적정 최대값(Q3)`를 명시
- `오피스 공간 프로그램 구성 비율 추이`는 전체 1개 + 2025→2020 연도별 누적 가로 막대 구조로 변경
- 막대 내부에 비율 수치를 직접 표기하여 툴팁 의존도를 낮춤

## 3. 지표별 계산식 정정 이력

### 3.1 개괄

#### 3.1.1 연도별 공간 프로그램 구성 비율 추이

기존 문제:

- HTML은 연도 내 전체 면적을 합산한 뒤 프로그램 비율을 계산하고 있었음
- 이 방식은 큰 도면에 과도한 가중치를 부여하여 Tableau 수치와 불일치가 발생함

수정 내용:

- 각 도면에서 `프로그램 면적 / 도면 가용면적`을 먼저 계산
- 이후 같은 연도에 속한 도면별 비율의 평균을 산출

최종 기준:

- 연도별 비율 지표는 모두 `도면별 비율 평균` 방식 사용

### 3.2 업무공간 : 일반

#### 3.2.1 데스크 형태별 사용 비중

기존 문제:

- HTML은 전체 좌석 합산 기준 비율에 가까운 방식으로 계산되어 Tableau와 차이가 발생함

수정 내용:

- 도면별 데스크 형태 사용 인원 비중을 먼저 계산
- 이후 연도별 평균값으로 집계

최종 기준:

- `도면별 비중 산출 → 평균`

#### 3.2.2 데스크 너비별 사용 비중

기존 문제:

- 데스크 형태별 사용 비중과 동일한 오류 패턴 존재

수정 내용:

- 도면별 데스크 너비 사용 인원 비중을 먼저 계산한 뒤 평균 집계

최종 기준:

- `도면별 비중 산출 → 평균`

### 3.3 회의공간

#### 3.3.1 회의실 평균 개소

최종 기준:

- `프로그램='회의'`
- `오픈실='실'`
- `세부유형!='기타회의공간'`
- 회의실이 1개 이상 존재하는 도면만 평균 분모에 포함

#### 3.3.2 회의실 규모별 평균 개소

기존 문제:

- 규모별 지표를 계산할 때 `해당 규모가 있는 도면만` 분모에 넣어 평균이 과대 계산됨
- 예시: `150-199인` 필터 기준 `4인이하실` 평균이 HTML에서 약 5.0으로 부풀려짐

수정 내용:

- 규모별 평균 개소의 분모를 `회의실이 하나 이상 존재하는 도면 전체`로 통일
- 각 규모별 개수는 같은 공통 분모 기준으로 평균 처리

최종 기준:

- 규모별 평균 개소는 `subtype-positive drawing average`가 아니라 `meeting-bearing drawing average`

#### 3.3.3 오픈/실 회의 지표

최종 기준:

- `기타회의공간`은 전부 제외
- 오픈/실 구분 비중도 동일한 제외 기준을 적용

### 3.4 소셜공간

#### 3.4.1 소셜 공간 정의

기존 문제:

- HTML 내부에서 실제 데이터의 프로그램 값 `공용`이 아니라 화면용 개념인 `소셜`을 직접 기준으로 사용하는 부분이 있었음

수정 내용:

- 계산 로직은 모두 원천 데이터의 `공용` 프로그램 값을 사용
- 화면 표시만 `소셜공간` 또는 `소셜`로 표현

#### 3.4.2 소셜 공간 내 유형별 구성 비율

기존 문제:

- 소셜 내부 비중이 아니라 도면 전체 면적 대비 비중으로 계산되던 문제 존재

수정 내용:

- 각 도면에서 `세부유형 면적 / 공용 총면적`을 계산
- 이후 연도별로 도면 평균 비율을 산출

최종 기준:

- 분모는 반드시 `공용 총면적`
- 연도 집계는 `도면별 비율 평균`

### 3.5 지원공간

#### 3.5.1 폰부스 및 선택업무공간_실 평균 개소

최종 기준:

- 해당 공간이 존재하는 도면만 평균 분모에 포함

#### 3.5.2 OA 평균 개소

최종 기준:

- OA가 존재하는 도면만 평균 분모에 포함

비고:

- 이 기준은 사용자 의사결정에 따라 의도적으로 유지됨

### 3.6 업무공간 : 임원

#### 3.6.1 가구구성별 평균 면적

기존 문제:

- 실형/오픈형과 세부유형 매핑 오류로 일부 값이 사실상 0에 가깝게 계산됨

수정 내용:

- 임원 세부유형 매핑 테이블을 정비하여 실형/오픈형별 평균 면적이 실제 데이터와 일치하도록 수정

## 4. 2026-04-07 2차 정비: 업무공간 : 일반 탭 개편

### 4.1 상단 KPI 구조 개편

기존 구성:

- 평균 인당면적
- 자율좌석제 비중
- 그룹 업무 공간 보유
- 주요 데스크 형태

변경 후 구성:

- 오피스 공간 프로그램 중 `업무 공간` 비중
- 평균 인당 면적
- 주요 등간격(일자형)
- 주요 등간격(L자형)
- 주요 데스크 형태
- 주요 데스크 너비

### 4.2 인당 면적 카드 개편

- 전체 평균 인당 면적을 KPI로 노출
- 카드 내부에서는 연도 구분 없는 전체 값과 연도별 추세를 함께 제공
- 그래프 라벨이 x축과 겹치지 않도록 y축 범위와 데이터 라벨 위치 조정
- 커서를 올리지 않아도 값이 바로 보이도록 표기 방식 강화

### 4.3 자율좌석제 시행 비중 개편

- 전체 도면 기준 값은 그래프가 아니라 큰 수치 카드로 분리
- 연도별 추세는 별도 그래프로 제공
- 전체 값과 연도별 값이 한눈에 구분되도록 레이아웃 분리

### 4.4 데스크 형태별/너비별 사용 비중 개편

- `전체 도면 기준`과 `연도별 평균 추세`를 같은 지표 카드에서 함께 볼 수 있도록 구조 개편
- 차트 안에서 전체와 연도별 구간이 시각적으로 분리되도록 보조선 및 구획 추가
- 막대/라벨이 카드 밖으로 밀리지 않도록 데이터 라벨 위치 조정
- 수치를 툴팁 없이 읽을 수 있도록 값 표시 강화

### 4.5 팀장석 구성 유형 개편

- 전체 도면 기준은 도넛 차트로 제공
- 연도별 추세는 별도 그래프로 제공
- 값 레이블이 막대 밖으로 밀리는 문제를 줄이도록 표현 방식 조정

### 4.6 등간격 지표 개편

- 그래프 중심 구조를 `1순위 / 2순위 / 3순위` 카드형 구조로 변경
- 일자형, L자형을 분리하여 각각 상위 구간과 비중을 표시
- 가장 많이 선택된 구간뿐 아니라 2순위, 3순위도 함께 제시

등간격 구간 기준 변경:

- `1600이하`
- `1601-1750`
- `1751-1900`
- `1901-2050`
- `2051-2200`
- `2201이상`

최종 산출 기준:

- 도면 단위 대표 등간격 값(`등간격_일자형`, `등간격_L자형`)을 수집
- 각 값을 위 구간에 binning
- 구간별 도면 수 비중으로 순위를 산출

### 4.7 그룹 업무 공간 보유 비중 개편

- 전체 도면 기준 값은 큰 수치로 노출
- 연도별 값은 별도 그래프로 제공
- 그래프 없이 값만 보이도록 했던 임시 구조를 개선하여, 전체와 연도별 정보를 명확히 구분

### 4.8 삭제/정리된 구성

- `주요 데스크 채택 현황` 카드 삭제
- 중복되거나 정보 전달력이 낮은 카드 정리

## 5. 숫자 표기 원칙

최종 기준:

- KPI 숫자: 소수점 둘째 자리까지 표시
- 차트 내부 값: 소수점 둘째 자리까지 표시
- 비율 값: `%` 단위 포함 소수점 둘째 자리까지 표시
- 자연수 기반 히트맵/건수 표: 정수로 표시

## 6. 재발 방지 가이드

이번 정비를 통해 반복 확인된 주요 오류 패턴은 다음과 같다.

### 6.1 합산 후 비율 계산 오류

- 연도 전체 면적 합산 후 비율을 내면 큰 도면에 과도한 가중치가 실린다.
- 비율 추이형 지표는 반드시 `도면별 비율 → 평균` 순서인지 검증해야 한다.

### 6.2 평균 분모 오판

- `해당 subtype이 있는 도면만 평균`인지
- `해당 category가 있는 도면 전체를 평균`인지
- `전체 도면 평균`인지

위 세 가지를 PRD와 대시보드에서 동일하게 써야 한다.

### 6.3 표시용 라벨과 원천 데이터값 혼동

- 화면상 `소셜`로 보이더라도 데이터상 프로그램 값은 `공용`일 수 있다.
- 지표 계산은 반드시 원천 데이터 기준으로 수행하고, 화면 라벨은 별도로 관리해야 한다.

### 6.4 화면 표현 변경 시 계산식 영향 검토 필요

- 지표를 카드, 차트, 표 중 어떤 형태로 바꾸더라도 계산식 자체가 바뀌는 것은 아니다.
- UI 개편 전후에 산출 기준이 유지되는지 반드시 확인해야 한다.
