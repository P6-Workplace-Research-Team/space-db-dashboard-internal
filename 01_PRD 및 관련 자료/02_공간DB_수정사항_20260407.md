# 공간DB 대시보드 수정 PRD

작성일: 2026-04-07  
대상 산출물: 공간DB 대시보드 HTML, Tableau 대시보드 산출 기준 정렬

## 1. 문서 목적

본 문서는 기존 `오피스 공간 데이터베이스 PRD` 및 Tableau 대시보드 검토 결과를 반영하여, 현재 배포 예정인 HTML 대시보드의 지표 산출 기준을 재정의하고 수정 사항을 기록하기 위한 문서이다.

이번 수정의 핵심 목적은 다음과 같다.

- HTML 대시보드와 Tableau 대시보드 간 산출 방식 불일치 해소
- 지표별 분모/분자 정의 명확화
- 연도 추이, 평균 개소, 구성 비율 등의 계산 기준 표준화
- 배포용 HTML 대시보드의 표시 형식 통일

## 2. 적용 범위

본 문서는 다음 대상에 적용된다.

- `공간DB_대시보드.html`
- `★ [2025 공간DB] Dashboard_260128.twbx`

## 3. 최종 적용 원칙

### 3.1 연도 추이 표시 원칙

- 연도 추이 차트에는 `2020~2025` 데이터만 표시한다.
- `2017`, `2018`, `2019` 도면은 모수가 작아 추세 해석 왜곡 가능성이 있으므로 시각화에서는 제외한다.
- 단, 원천 데이터나 필터 자체에서 해당 연도를 제거하는 것은 아니며, 표시 영역에서만 제외한다.

### 3.2 회의 지표 원칙

- 회의 관련 지표는 모두 `기타회의공간`을 제외한다.
- 회의실 여부는 `프로그램 = 회의`이면서 `오픈실 = 실`이고 `세부유형 != 기타회의공간`인 공간을 기준으로 판단한다.
- 전체 회의실 평균 개소는 `회의실이 1개 이상 존재하는 도면만` 평균에 포함한다.

### 3.3 지원공간 평균 산출 원칙

- 폰부스, 선택업무공간실, OA 등 지원공간 평균 개소는 `해당 공간이 존재하는 도면만` 평균에 포함한다.
- 해당 공간이 없는 도면은 평균 계산에서 제외한다.

### 3.4 비율 지표 산출 원칙

- 연도별 구성 비율 차트는 원칙적으로 `연도 합산 후 비율`이 아니라 `도면별 비율을 먼저 계산한 뒤 평균`하는 방식으로 산출한다.
- 즉, 큰 도면에 가중치가 과도하게 실리지 않도록 `도면 단위 평균`을 기본 원칙으로 한다.

### 3.5 표시 형식 원칙

- 대시보드 내 모든 수치 표시는 소수점 둘째 자리까지 노출한다.
- 퍼센트, KPI 수치, 데이터 라벨, 툴팁 등 모든 사용자 노출 숫자 형식에 동일하게 적용한다.

## 4. 수정 사항 상세

## 4.1 개괄 섹션

### 4.1.1 연도별 공간 프로그램 구성 비율 추이

기존 문제:

- HTML은 연도별 전체 면적을 합산한 뒤 프로그램 비율을 계산하고 있었다.
- 이 방식은 대형 도면 비중이 과도하게 커져 Tableau 수치와 달라졌다.

최종 기준:

- 각 도면에서 `프로그램 면적 / 도면 가용면적` 비율을 먼저 계산한다.
- 이후 동일 연도 도면들의 비율 평균으로 연도별 값을 산출한다.

기대 효과:

- 인원수 구간, 가용면적 구간 등 필터 적용 시 Tableau와 동일한 방식으로 동작한다.
- 특정 대형 도면의 영향으로 구성비가 왜곡되는 문제를 방지한다.

## 4.2 업무 섹션

### 4.2.1 데스크형태별 사용 비중

기존 문제:

- HTML이 연도별 총 좌석 수를 합산한 뒤 데스크형태 비율을 계산하고 있었다.

최종 기준:

- 도면별 데스크형태 비율을 계산한 뒤, 연도별 평균값으로 집계한다.

### 4.2.2 데스크너비별 사용 비중

기존 문제:

- HTML이 합산 후 비율 방식으로 계산되어 Tableau와 차이가 발생했다.

최종 기준:

- 도면별 너비 구간 비율을 먼저 계산한 뒤, 연도별 평균값으로 산출한다.

## 4.3 회의 섹션

### 4.3.1 회의실 평균 개소

최종 기준:

- `기타회의공간` 제외
- `회의실이 존재하는 도면만` 평균에 포함

정의:

- 회의실 존재 도면 = `프로그램 = 회의` AND `오픈실 = 실` AND `세부유형 != 기타회의공간`

### 4.3.2 회의실 규모별 평균 개소

기존 문제:

- HTML은 각 규모별 회의실이 실제로 존재하는 도면만 분모로 사용하고 있었다.
- 예: `4인이하실` 평균을 계산할 때 `4인이하실이 있는 도면만` 평균에 포함하여 값이 과대 산출되었다.
- 이로 인해 `150-199인` 필터 기준 HTML이 약 `5.0` 수준으로 표시되었으나, Tableau는 `1.98`이었다.

원인:

- 전체 회의실 평균 개소와 규모별 회의실 평균 개소의 분모 기준이 서로 달랐다.
- 규모별 평균에서도 공통 분모를 써야 하는데, 세부유형별 존재 도면만 분모로 잡혀 있었다.

최종 기준:

- 규모별 회의실 평균 개소는 `해당 규모 회의실이 있는 도면만`이 아니라, `회의실이 1개 이상 있는 도면 전체`를 공통 분모로 사용한다.
- 각 도면의 규모별 회의실 개수를 계산한 뒤 그 값을 평균낸다.

예시:

- `150-199인` 필터에서 `4인이하실` 평균 개소는 `회의실이 있는 도면 전체` 기준 평균으로 계산해야 하며, 이 기준에서 Tableau와 동일한 `1.98` 수준으로 일치한다.

### 4.3.3 오픈형 회의공간 비율

최종 기준:

- 회의 프로그램 중 `기타회의공간`은 제외한다.
- 오픈/실 구분은 제외 조건 적용 후 계산한다.

## 4.4 소셜 섹션

### 4.4.1 소셜 공간 정의

기존 문제:

- HTML 일부 로직이 데이터상 실제 프로그램 값인 `공용`이 아닌 `소셜`을 기준으로 계산하고 있었다.

최종 기준:

- 데이터 원천의 프로그램 값은 `공용`을 사용한다.
- 사용자 화면 라벨은 필요 시 `소셜`로 표시한다.

### 4.4.2 연도별 소셜공간 유형 구성 비율

사용자 확정 기준:

- `소셜 공간 안에서 세부유형별 면적 비중`으로 본다.

최종 기준:

- 각 도면에서 `세부유형 면적 / 소셜(공용) 총면적`을 계산한다.
- 이후 연도별로 도면 평균을 낸다.

주의:

- 도면 전체 면적을 분모로 사용하지 않는다.
- 연도별 합산 후 비율 방식도 사용하지 않는다.

### 4.4.3 탕비/라운지 관련 집계

최종 기준:

- 탕비, 라운지 관련 세부유형 집계는 모두 `공용` 프로그램 내에서 계산한다.

## 4.5 지원 섹션

### 4.5.1 폰부스 + 선택업무공간실 평균 개소

최종 기준:

- `폰부스 또는 선택업무공간실이 존재하는 도면만` 평균에 포함한다.

### 4.5.2 폰부스 평균 개소

최종 기준:

- `폰부스가 존재하는 도면만` 평균에 포함한다.

### 4.5.3 선택업무공간실 평균 개소

최종 기준:

- `선택업무공간실이 존재하는 도면만` 평균에 포함한다.

### 4.5.4 OA 평균 개소

최종 기준:

- `OA가 존재하는 도면만` 평균에 포함한다.

비고:

- 해당 방식은 사용자의 명시적 요청에 따른 기준이다.

## 4.6 임원 섹션

### 4.6.1 실형/오픈형 가구구성별 평균 면적

기존 문제:

- 세부유형 매핑이 잘못되어 일부 값이 사실상 0에 가깝게 계산되고 있었다.

최종 기준:

- 실형과 오픈형 각각에 대해 올바른 세부유형 매핑 테이블을 적용한다.
- 세부유형별 집계가 실제 데이터 분류와 일치하도록 수정한다.

## 4.7 숫자 포맷

최종 기준:

- 모든 KPI 숫자: 소수점 둘째 자리 표시
- 모든 퍼센트 값: 소수점 둘째 자리 표시
- 차트 라벨: 소수점 둘째 자리 표시
- 사용자 노출 수치 전반: 소수점 둘째 자리 표시

## 5. 오류 유형 분석 및 재발 방지 가이드

이번 검토에서 확인된 주요 오류 패턴은 아래와 같다.

### 5.1 합산 후 비율 계산 오류

설명:

- 연도별 또는 구간별 데이터에서 전체 합을 먼저 구한 뒤 비율을 계산하면, 면적이 큰 도면이나 좌석 수가 많은 도면에 과도한 가중치가 부여된다.

예방 원칙:

- 구성 비율 추이 지표는 우선 `도면별 비율`을 계산하고, 이후 평균을 집계한다.

적용 대상:

- 공간 프로그램 구성 비율
- 데스크형태 비율
- 데스크너비 비율
- 소셜 세부유형 구성 비율

### 5.2 세부유형 존재 도면만 분모로 잡는 오류

설명:

- 세부유형별 평균 개소를 계산할 때, 그 세부유형이 존재하는 도면만 분모로 잡으면 평균이 과대 산출될 수 있다.

예방 원칙:

- 지표 목적에 따라 분모를 명확히 정의한다.
- 회의실 규모별 평균 개소는 `회의실이 존재하는 도면 전체`를 분모로 사용한다.
- 지원공간 평균 개소는 사용자 요구사항에 따라 `해당 공간이 존재하는 도면만` 분모로 사용한다.

### 5.3 코드값과 화면 라벨 혼용 오류

설명:

- 원천 데이터의 실제 프로그램 코드값과 화면 표기명이 다를 경우, 계산 로직에서 표기명을 직접 사용하면 집계 누락이 생길 수 있다.

예방 원칙:

- 계산 로직은 반드시 원천 데이터 코드값 기준으로 작성한다.
- 화면 라벨은 별도 매핑을 통해 표시한다.

예시:

- 원천 데이터 코드값: `공용`
- 화면 라벨: `소셜`

## 6. HTML 반영 결과

`공간DB_대시보드.html`에는 다음 사항이 반영되었다.

- 연도 추이 차트 표시 연도 `2020~2025` 제한
- 회의 지표에서 `기타회의공간` 제외
- 회의실 평균 개소 기준 수정
- 회의실 규모별 평균 개소 분모 수정
- 지원공간 평균 개소 기준 수정
- 개괄/업무/소셜 비율 지표를 도면별 평균 방식으로 수정
- 소셜 프로그램 계산 기준을 `공용`으로 수정
- 임원 섹션 세부유형 매핑 수정
- 숫자 포맷 소수점 둘째 자리 통일

## 7. 검증 결과 요약

검증 방식:

- HTML 내장 데이터 기준 계산식 재검산
- Tableau 워크북 계산식 구조와 로직 비교
- 주요 필터 구간 샘플 검증

주요 확인 사항:

- `150-199인` 필터 기준 `4인이하 회의실 평균 개소`가 Tableau 기준 `1.98`과 정렬되도록 수정됨
- 연도별 공간 프로그램 구성 비율 추이가 도면별 평균 방식으로 수정됨
- 지원공간 평균 개소가 `보유 도면 기준`으로 계산되도록 수정됨
- 숫자 표기가 모두 소수점 둘째 자리까지 표시되도록 수정됨
